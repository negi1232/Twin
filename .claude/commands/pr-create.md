# プルリクエスト作成

現在のブランチの GitHub Pull Request を作成します。

## 手順

1. 未コミットの変更がないか確認（あれば警告）
2. 現在のブランチ名からタイプを判定し、ベースブランチを決定:
   - `feature/*` → ベース: `develop`
   - `hotfix/*` → ベース: `main`
   - `release/*` → ベース: `main`
3. 現在のブランチを origin に push（未 push の場合）
4. ベースブランチからの差分コミットログを取得
5. Pull Request を作成:
   - タイトル: ブランチ名から生成（ハイフンをスペースに、先頭大文字）
   - 本文: コミットログからの変更概要 + テスト計画セクション
   - ベース: 自動判定したベースブランチ
6. PR の URL をユーザーに返す

## ルール
- ブランチ命名規則からベースブランチを自動判定する
- PR の本文に「テスト計画」セクションを含める
- 未コミットの変更がある場合は警告するがブロックしない
