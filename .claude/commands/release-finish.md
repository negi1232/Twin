# リリースブランチ完了

release ブランチを main と develop の両方にマージし、リリースタグを作成します。

## 手順

1. 現在のブランチが `release/*` であることを確認
2. `npm test` を実行し、全テストがパスすることを確認
3. `npm run lint` を実行し、エラーがないことを確認
4. ブランチ名からバージョンを取得（例: `release/1.0.0` → `1.0.0`）
5. `main` をチェックアウトし、`--no-ff` で release をマージ
6. マージコミットにタグを付与: `v<バージョン>`（例: `v1.0.0`）
7. `develop` をチェックアウトし、`--no-ff` で release をマージ
8. release ブランチをローカルで削除
9. 完了を報告し、両ブランチとタグの push をリマインドする

## ルール
- `release/*` ブランチでのみ実行可能
- テストが通らなければマージしない
- 必ず `--no-ff` マージ戦略を使用
- タグの形式: `v<バージョン>`
- main と develop の**両方**にマージする
- 自動で push はしない
